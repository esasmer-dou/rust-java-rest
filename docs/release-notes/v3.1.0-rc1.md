# v3.1.0-rc1 - Rust I/O Plane Performance Preview

This is a release candidate, not a stable production claim. The product direction is now explicit:
Java owns business logic and application composition; Rust owns the HTTP I/O plane, bounded native
memory, selected serialization-heavy paths, large/static response handling, and backpressure.

## What's New

### Install and Packaging

- Published Maven package: `com.reactor:rust-java-rest:3.1.0-rc1`.
- Published GitHub Release assets for manual/runtime distribution:
  `rust-java-rest-3.1.0-rc1.jar`, `rust-java-rest-3.1.0-rc1-sources.jar`,
  `rust-java-rest-3.1.0-rc1-javadoc.jar`, `rust_hyper-windows-x64.dll`, and
  `librust_hyper-linux-x64.so`.
- The Maven/JAR package embeds platform native libraries, so normal users do not need to copy DLL/SO
  files manually:
  `native/windows-x64/rust_hyper.dll` and `native/linux-x64/librust_hyper.so`.
- Custom native loading is still supported with `-Drust.lib.path=/absolute/path/to/library`.
- Native ABI version check was added. Java now fails fast if an old DLL/SO is loaded with the new Java
  runtime.

### New Response Types

- `RawResponse` lets handlers return pre-serialized bytes without building a Java DTO response graph.
- `RawResponse.text(...)`, `RawResponse.bytes(...)`, and `RawResponse.json(...)` cover direct text,
  binary, and JSON response bodies.
- `RawResponse.registeredJson(...)` registers immutable JSON bytes in Rust once and lets hot read-heavy
  routes return a small native response id.
- `RawResponse.nativeJson(nativeId)` lets Java return a previously registered native response directly.
- `FileResponse.of(path)` and `FileResponse.download(path, fileName, contentType)` let Java return a
  file descriptor-style response while Rust streams the file. File bytes do not cross Java/JNI.
- `@MaxRequestBodySize(bytes)` and `@MaxResponseSize(bytes)` add per-route body limits on top of global
  runtime limits.

### Direct JSON Writer Path

- `JsonBufferWriter` lets hot handlers write JSON directly into a native direct `ByteBuffer`.
- `DirectJsonWriter<T>` defines the generated/build-time writer contract:
  `int write(T value, ByteBuffer out, int offset)`.
- Direct writer methods return a positive written length on success or a negative required length when
  Rust should retry with a larger response buffer.
- `@DirectQueryInt("param")` lets Rust parse selected query integer parameters and pass primitive `int`
  values into Java, avoiding Java query `String` allocation on selected hot routes.
- This path is intended for serialization-heavy endpoints where building a large DTO graph per request
  is the real memory/latency bottleneck.

### WebSocket Runtime Behavior

- `WebSocketSession.sendText(...)`, `sendBinary(...)`, `close()`, and `close(code, reason)` now go
  through a Rust-side session registry instead of a stale instance native pointer.
- WebSocket sessions expose `getPathParams()` and `getQueryParams()` for path/query-aware handlers.
- Outbound WebSocket sends are bounded by native queue capacity, max frame size, and send timeout.
- Slow consumers are rejected/closed instead of allowing unbounded memory growth.
- `WebSocketBroadcaster` continues to provide broadcast, room broadcast, binary broadcast, and filtered
  broadcast APIs, now backed by bounded native send behavior.

### Built-in Metrics and Diagnostics

- `GET /metrics` returns native plus Java metrics in Prometheus text format.
- `GET /metrics/summary` returns a compact JSON summary for uptime, heap, request count, and active
  connections.
- `GET /metrics/reset` resets Java and native metrics for benchmark runs.
- `GET /diagnostics/memory` returns one JSON report with JVM heap/non-heap, direct buffer pools, native
  memory diagnostics, response pool counters, in-flight bytes, and native metrics.
- `GET /diagnostics/native/trim` asks the native runtime to release/trim retained native memory where
  supported, then returns native memory diagnostics.

### Runtime Properties Users Can Tune

- HTTP limits: `reactor.rust.http.max-request-body-bytes`,
  `reactor.rust.http.max-response-body-bytes`, `reactor.rust.http.max-inflight-body-bytes`,
  `reactor.rust.http.max-inflight-response-bytes`, `reactor.rust.http.max-connections`,
  `reactor.rust.http.max-request-header-bytes`, and `reactor.rust.http.max-request-headers`.
- HTTP timeout/connection behavior: `reactor.rust.http.header-read-timeout-ms`,
  `reactor.rust.http.request-body-timeout-ms`, `reactor.rust.http.idle-timeout-ms`,
  `reactor.rust.http.keep-alive-enabled`, and `reactor.rust.http.http1-only-enabled`.
- WebSocket limits: `reactor.rust.websocket.max-frame-bytes`,
  `reactor.rust.websocket.outbound-queue-capacity`, and
  `reactor.rust.websocket.send-timeout-ms`.
- JNI/backpressure: `reactor.rust.jni.workers` and `reactor.rust.jni.queue-capacity`.
- Native response pooling: `reactor.rust.response-pool.small-capacity`,
  `reactor.rust.response-pool.medium-capacity`, `reactor.rust.response-pool.large-capacity`, and
  `reactor.rust.response-pool.huge-capacity`.
- Native cache bounds: `reactor.rust.native-cache.max-entries`,
  `reactor.rust.native-cache.max-bytes`, and `reactor.rust.native-cache.ttl-ms`.
- Runtime sizing: `reactor.rust.runtime.worker-threads`,
  `reactor.rust.runtime.max-blocking-threads`, and `reactor.rust.runtime.thread-stack-bytes`.
- JSON writer retention: `reactor.rust.json.writer-initial-bytes` and
  `reactor.rust.json.writer-retain-max-bytes`.
- Logging controls: `reactor.rust.log.level` and `reactor.rust.java.log.level`.

### Benchmark and Deployment Profiles

- Added container benchmark support for comparing this framework against Spring Boot under the same
  endpoint set and container limits.
- Added repeated/randomized benchmark support and a custom Rust load probe for consistent p50/p95/p99,
  error-rate, throughput, and memory snapshots.
- Added memory proof runs around warmup, load, idle, and native trim phases.
- Added low-RSS, ultra-low-RSS, micro-RSS, and throughput runtime profile presets for different
  production goals. These profiles intentionally trade memory, queue depth, and throughput differently.

## Positioning

BEST use cases:
- Small JSON APIs where Java handlers should avoid Spring Boot runtime overhead.
- Read-heavy or precomputed/raw JSON responses.
- Large file/static/export responses served by Rust without moving body bytes through Java.
- Services that need bounded RSS and predictable overload behavior.

ACCEPTABLE use cases:
- Dynamic JSON endpoints where business logic remains Java and response serialization uses direct
  writer/generated-style paths.
- WebSocket workloads with bounded outbound queues and explicit slow-consumer handling.

ANTI-PATTERN:
- Claiming universal "5x faster than Spring Boot" for every endpoint.
- Building large Java DTO object graphs per request and expecting Rust I/O alone to remove that cost.
- Increasing request/response limits without in-flight byte caps.

## Key Changes

- Added bounded JNI worker queue and explicit backpressure behavior.
- Added configurable request body, response body, in-flight byte, header, timeout, and keep-alive limits.
- Added response buffer pooling, response body ownership improvements, and native memory diagnostics.
- Added `RawResponse`, `FileResponse`, static/native response registration, and large response paths.
- Added WebSocket outbound send registry with bounded queues and path/query support.
- Added low-overhead metrics and hot-path logging cleanup.
- Added low-RSS, ultra-low-RSS, micro-RSS, and throughput runtime benchmark profiles.
- Added direct JSON writer APIs and generated-style writer/parser prototypes for hot DTO paths.
- Added `@DirectQueryInt` for primitive query parameter passing from Rust to Java without Java query
  `String` allocation on selected direct routes.
- Rebuilt Windows `rust_hyper.dll` and Linux `librust_hyper.so` for native ABI 9.

## Production Gate

This RC can be used for preview and controlled benchmark validation. Before stable release:

- Run benchmark with `RepeatCount >= 3` and publish median/p95 across runs.
- Tune JNI queue/backpressure to reduce or eliminate low-rate `503` at c=1000.
- Restore CI/CD matrix for Windows DLL and Linux glibc `.so` builds.
- Align Rust native source repository access with Java framework release artifacts.
- Document `RawResponse`, `FileResponse`, `@DirectQueryInt`, runtime profiles, metrics, and limits.
